  1. HTML is an acronym for HyperText Markup Language.
    • Hypertext is text which can link to other text.
    • A markup language is a way of annotating plain-text with instructions for a web-browser
  2. To write html means to annotate plain-text with tags.

    If you want to tell a browser how to display your text, you cannot write:

    Hey browser, "having fun" should be italicized!

    You'd have to define "having fun" as an emphasized element, by wrapping it in tags:

    <em> having fun </em>

    Most elements have a start and an end tag.

    start tag: <tagname>

    end tag: </tagname>

    These define the start and the end of a certain element

    <em>start-tag having fun content</em>end-tag

    You could say tags are hidden messages for the browser.

  3. Tags define elements. Elements make up a webpage.

    <html>

    <title> This is a simplified* Webpage </title>

    <h1>

    All Webpages Are Made Of Elements

    </h1>

    <strong> Different elements are displayed differently <strong>

    <p> Some elements cannot contain anything.. they are void. The br <br>
    (line break) element for example. This is why it is <em> just </em> a start tag. </p>

    <p> All elements that can contain text, can also contain other elements. </p>

    </html>

    * This example-page is over-simplified. It works, although it is not actually valid. More on this in point 5.

    Parents, Children, Siblings and Roots? These are terms which describe how elements relate to each other:
    • A parent is an element that contains something.
    • A child is an element that is inside a parent element.
    • Siblings are elements that have the same parent.
    • A root element has no parent.
  4. All elements can have attributes.
    • Attributes give the browser more information about an element.
    • They are always in the start tag of an element.
    • Usually they consist of a name and a value.

    <tagname  space attribute="value">...

    <img image-tag srcsource="cat.jpgvalue">

    Some elements need attributes.

    The html element contains all elements of a webpage. It should have a language attribute.

    <html lang="en">
    . . .
    </html>

    The anchor element is used to link to other websites. It needs a href attribute.
    The value is usually a URL

    <a href="https://wikipedia.com">wikipedia</a>

    The image element needs a source attribute which specifies the path to the image.
    It should also have an alternative-text attribute for screen readers.
    Often times it has a width (and/or height) attribute aswell.

    <img src="cat.jpg" alt="cat close-up" width="250">

  5. Most webpages use a similar boilerplate.

    Here are some elements every webpage should have
    A new html document usually starts with a boilerplate.
    Feel free to copy/paste this one :)

    <!DOCTYPE html> This is a doctype declaration. It tells the browser that a document is written in HTML 5 (the current standard)
    <html lang="en"> start of the html element + language attribute. The html element is the root element - it contains the whole webpage, except the doctype declaration.
      <head> start of the head element, which contains all 'invisible' metadata. There can only be one head in a document.
        <title>TITLE</title> The page-title. It is for the browser-tab and search engines.
        <meta charset="utf-8"> This defines the character set a browser uses to display the webpage. It makes using spëcîål characters possible.
        <meta name="viewport" content="width=device-width, initial-scale=1.0"> This stops mobile phones from automatically zooming out of your page
        <link rel="stylesheet" href="style.css"> This is how you reference an external stylesheet. If you don't have one you don't need this line.
        <style> Within this element you can declare styles using CSS. If you use an external style-sheet you don't need it.
        * { box-sizing: border-box; } This is a line of CSS. The CSS-working group admitted that not choosing this as a default was a mistake.
        </style>
       </head>
      <body> The body element contains all of a webpage's content. There can only be one body in a document.
